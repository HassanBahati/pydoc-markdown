    MarkdownRenderer Objects

    @dataclasses.dataclass
class MarkdownRenderer(Renderer, SinglePageRenderer, SingleObjectRenderer)

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L40)

    Produces Markdown files. This renderer is often used by other renderers, such as MkdocsRenderer and HugoRenderer. It provides a wide variety of options to customize the generated Markdown files.

    Options

    filename

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L51)

    The name of the file to render to. If no file is specified, it will render to stdout.

    encoding

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L55)

    The encoding of the output file. This is ignored when rendering to stdout.

    insert_header_anchors

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L59)

    If enabled, inserts anchors before Markdown headers to ensure that links to the header work. This is enabled by default.

    html_headers

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L63)

    Generate HTML headers instead of Mearkdown headers. This is disabled by default.

    code_headers

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L67)

    Render names in headers as code (using backticks or <code> tags, depending on html_headers). This is enabled by default.

    descriptive_class_title

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L71)

    Generate descriptive class titles by adding the word "Objects" after the class name. This is enabled by default.

    descriptive_module_title

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L75)

    Generate descriptivie module titles by adding the word "Module" before the module name. This is enabled by default.

    add_module_prefix

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L80)

    Add the module name as a prefix to class & method names. This module name is also rendered as code if code_headers is enabled. This is enabled by default.

    add_method_class_prefix

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L85)

    Add the class name as a prefix to method names. This class name is also rendered as code if code_headers is enabled. This is enabled by default.

    add_member_class_prefix

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L89)

    Add the class name as a prefix to member names. This is enabled by default.

    add_full_prefix

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L93)

    Add the full module name as a prefix to the title of the header. This is disabled by default.

    sub_prefix

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L97)

    If add_full_prefix is enabled, this will result in the prefix to be wrapped in a <sub> tag.

    data_code_block

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L101)

    Render the definition of data members as a code block. This is disabled by default.

    data_expression_maxlength

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L106)

    Max length of expressions. If this limit is exceeded, the remaining characters will be replaced with three dots. This is set to 100 by default.

    classdef_code_block

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L110)

    Render the class signature as a code block. This includes the "class" keyword, the class name and its bases. This is enabled by default.

    classdef_with_decorators

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L113)

    Render decorators before class definitions.

    signature_python_help_style

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L117)

    Render classdef and function signature blocks in the Python help() style.

    signature_code_block

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L122)

    Render the function signature as a code block. This includes the "def" keyword, the function name and its arguments. This is enabled by default.

    signature_in_header

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L125)

    Render the function signature in the header. This is disabled by default.

    signature_with_vertical_bar

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L128)

    Render the vertical bar '|' before function signature. This is enabled by default.

    signature_with_def

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L132)

    Include the "def" keyword in the function signature. This is enabled by default.

    signature_class_prefix

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L137)

    Render the class name in the code block for function signature. Note that this results in invalid Python syntax to be rendered. This is disabled by default.

    signature_with_decorators

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L140)

    Render decorators before function definitions.

    render_typehint_in_data_header

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L143)

    Render type hints for data elements in the header.

    code_lang

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L147)

    Add the string "python" after the backticks for code blocks. This is enabled by default.

    render_page_title

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L150)

    Render title of page at the beginning of the file.

    render_toc

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L153)

    Render a table of contents at the beginning of the file.

    render_toc_title

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L156)

    The title of the "Table of Contents" header.

    toc_maxdepth

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L159)

    The maximum depth of the table of contents. Defaults to 2.

    render_module_header

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L162)

    Render module headers. This is enabled by default.

    render_module_header_template

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L165)

    Custom template for module header.

    docstrings_as_blockquote

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L168)

    Render docstrings as blockquotes. This is disabled by default.

    use_fixed_header_levels

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L172)

    Use a fixed header level for every kind of API object. The individual levels can be defined with header_level_by_type.

    header_level_by_type

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L175)

    Fixed header levels by API object type.

    source_linker

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L186)

    A plugin that implements the SourceLinker interface to provide links to the source code of API objects. If this field is specified, the renderer will place links to the source code in the generated Markdown files.

    source_format

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L195)

    Allows you to override how the "view source" link is rendered into the Markdown file if a source_linker is configured. The default is [[view_source]]({url}).

    escape_html_in_docstring

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L198)

    Escape html in docstring. Default to False.

    render_novella_anchors

    [[view_source]](https://github.com/NiklasRosenstein/pydoc-markdown/blob/(HEAD/docs/../src/pydoc_markdown/contrib/renderers/markdown.py#L201)

    Render Novella @anchor tags before headings.
